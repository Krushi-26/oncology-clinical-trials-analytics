Ingestion Layer

api_client.py
  • Sends HTTP GET requests to ClinicalTrials.gov endpoint

  • Configurable filters: condition · status · date range
  • Returns raw JSON payload to pipeline orchestrator

03–05
Core ETL Engine
🔄 Orchestrator
⚙️
run_pipeline.py
  • Retrieves watermark from etl_run_logs
  • Sequences extract → transform → load
  • Triggers transformation on raw data
  • Writes run metadata on completion
🔧 Transform
🔄
transform.py
  • Flattens deeply nested JSON
  • Extracts structured analytical fields
  • Parses & normalises date formats
  • Handles nulls, type casting, defaults
⏱ Incremental
🔍
Watermark Logic
  • Compare last_update_posted vs watermark
  • Load only new / updated records
  • Skip entirely unchanged records
  • Advance watermark on success
↓ Cleaned, Filtered Records
06
Load Layer
📥 Upsert Strategy
⬆️
load.py
  • Batch INSERT into MySQL clinical_trials table
  • ON DUPLICATE KEY UPDATE — safe idempotent upsert
  • Deduplication enforced on nct_id primary key
  • Transaction commit on success · full rollback on failure
↓ Persisted to Storage
